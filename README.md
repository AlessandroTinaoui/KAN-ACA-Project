# KAN ACA Project

This repository contains Kolmogorov-Arnold Network (KAN) experiments for conic-section classification and a few tabular baselines. The main conic experiments are:

- a single-task KAN baseline;
- a class-incremental continual learning KAN;
- a pruning-first KAN experiment.

The code is organized as a Python package under `src/`, while experiment settings are stored in TOML files under `configs/`.

## Requirements

Use Python 3.11 or newer.

Main Python dependencies:

```text
torch
pykan
numpy
pandas
matplotlib
tqdm
scikit-learn
```

Recommended setup:

```powershell
python -m venv .venv
.\.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -e .
pip install torch pykan numpy pandas matplotlib tqdm scikit-learn
```

If you already have a working `.venv`, activate it before running the experiments.

## Folder Management

### `configs/`

Important conic configs:

- `configs/conic/baseline.toml`: single-task KAN baseline.
- `configs/conic/continual.toml`: standard continual learning order.
- `configs/conic/continual_reversed.toml`: reversed continual learning order.
- `configs/conic/pruning.toml`: pruning-first experiment.

### `dataset/`

The conic experiments use:

```text
dataset/Conic-Section_dataset.csv
```

The dataset generator is:

```text
dataset/Conic_Shapes_Generator.py
```

### `src/kan_models/`

Contains the Python package.

The conic implementation is split into:

- `baseline.py`: single-task all-classes training.
- `pruning.py`: probe-based pruning-first experiment.
- `continual/experiment.py`: continual learning orchestration.
- `continual/training.py`: continual training loop, balanced sampler, LwF, and early stopping.
- `data.py`: dataset loading, stratified split, and standardization.
- `modeling.py`: model creation and evaluation helpers.
- `plotting.py`: plots, confusion matrices, and edge-function exports.

### Output folders

The experiments write plots, CSV files, JSON configs, and model checkpoints in the project root.

Common generated folders:

- `conic_plots/`: baseline plots.
- `continual_plots/`: standard continual learning plots.
- `continual_plots_reversed/`: reversed continual learning plots.
- `pruning_plots/`: pruning-first plots.
- `artifacts/`: artifacts for tabular experiments.

These outputs are generated by the scripts and can be deleted when you want a clean run.

## Running the Conic Experiments

Run all commands from the repository root:

```powershell
cd KAN-ACA-Project
.\.venv\Scripts\activate
```

### 1. Single-Task Baseline

This trains one KAN on all conic classes at the same time.

```powershell
python src/kan_models/models/conic/main.py
```

Equivalent explicit config command:

```powershell
python src/kan_models/models/conic/main.py --config configs/conic/baseline.toml
```

### 2. Continual Learning

This trains the KAN stage by stage using the class order defined in `continual.toml`.

```powershell
python src/kan_models/models/conic/continual/main.py --variant standard
```

You can also run the standard version without arguments:

```powershell
python src/kan_models/models/conic/continual/main.py
```

### 3. Continual Learning - Reversed Order

This runs the same continual learning code, but with the class order defined in `continual_reversed.toml`.

```powershell
python src/kan_models/models/conic/continual/main.py --variant reversed
```

Equivalent explicit config command:

```powershell
python src/kan_models/models/conic/continual/main.py --config configs/conic/continual_reversed.toml
```

### 4. Pruning-First Experiment

This trains a larger probe model, uses KAN attribution scores to select important hidden nodes, then retrains progressively smaller pruned models.

```powershell
python src/kan_models/models/conic/pruning.py
```

Equivalent explicit config command:

```powershell
python src/kan_models/models/conic/pruning.py --config configs/conic/pruning.toml
```

Main outputs:

```text
pruning_metrics.csv
high_loss_examples.csv
pruning_plots/
```
